# 고민 주제

> **User Entity에 어떤 필드를 넣고, 생성자는 어떤 값을 받아야 할까?**

---

# 결론

* User는 카카오/GitHub 소셜 로그인 정보 + GitHub 자동 커밋 연동 정보를 함께 관리하는 Entity.
* 생성자는 가입 시점에 정해지는 값(`email`, `nickname`, `provider`, `providerUserId`)만 받는다.

---

# User의 역할

* 카카오/GitHub 소셜 로그인 사용자 관리 (일반 회원가입 없음)
* GitHub 저장소 자동 커밋 연동 정보(PAT, repo) 관리
* Retrospective, KptNote 등 다른 Entity의 부모 (User 1 : N)

---

# 필드별로 왜 필요한지

* **`email`**: 표시용 이메일. `provider`가 다르면 같은 이메일이어도 별개 유저로 취급하기로 해서 UNIQUE 아님.
* **`nickname`**: 사용자에게 보여지는 이름. 최초 로그인 시 GitHub는 username, 카카오는 프로필 이름을 기본값으로 사용.
* **`provider`**: 소셜 로그인 제공자(`KAKAO`/`GITHUB`). 계정 병합을 안 하기로 해서 유저는 평생 하나만 가짐.
* **`providerUserId`**: provider가 발급한 고유 ID (카카오 회원번호, GitHub user id 등).
* **`repoOwner`/`repoName`/`githubPat`**: GitHub 자동 커밋 연동 정보. 연동 전엔 전부 null.
* **`githubUploadEnabled`**: PAT/repo가 등록돼 있어도 자동 커밋 자체를 껐다 켰다 할 수 있어야 해서 별도 boolean으로 둠.

---

# 생성자에 어떤 값을 받을지

가입 시점에 실제로 정해지는 값(`email`, `nickname`, `provider`, `providerUserId`)만 받는다.

* `id`: DB가 `AUTO_INCREMENT`로 채번하는 값이라 저장 전엔 알 수 없어서 제외.
* `repoOwner`/`repoName`/`githubPat`: 가입 시점이 아니라 나중에 설정 화면에서 GitHub 연동할 때 채워지는 값이라 제외.
* `githubUploadEnabled`: `NOT NULL DEFAULT FALSE`라 필드 선언 시 기본값(`= false`)으로 처리하고 생성자에서 안 받음.
* `createdAt`/`updatedAt`: JPA Auditing이 채워주는 값이라 생성자와 무관 (자세한 내용은 `BaseTimeEntity.md` 참고).

Entity의 생성자/Setter 정책(`@Setter` 개방, protected 기본 생성자)은 모든 Entity에 공통이라 `Entity 공통 컨벤션.md`에 정리함.
