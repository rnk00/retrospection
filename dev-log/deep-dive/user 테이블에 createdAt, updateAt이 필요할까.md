## createdAt / updatedAt 컬럼

**없어도 서비스 자체는 동작하지만, 실제 서비스 개발에서는 대부분 추가하는 것을 추천한다.**

데이터가 생성되고 수정되는 시간을 기록하는 컬럼으로, 단순한 기록 용도를 넘어 **데이터 관리, 통계, 장애 추적, 서비스 기능 구현**에 활용된다.

---

### createdAt

데이터가 **언제 생성되었는지 확인하기 위해** 사용하는 컬럼이다.
ex) 사용자: 회원가입 날짜 확인

또한 시간 기준의 기능을 만들 때 필요하다.
ex) 이번 달 가입한 사용자 수, 최근 7일 동안 작성된 회고 개수, 최신순으로 회고 목록 정렬

운영 중 문제가 발생했을 때도 유용하다.
ex) 특정 데이터에 문제가 생겼다면 언제 생성된 데이터인지, 특정 시점 이후 발생한 문제인지를 확인할 수 있어 데이터 추적과 장애 분석에 도움이 된다.

---

### updatedAt

데이터가 **마지막으로 수정된 시간을 기록하기 위해** 사용하는 컬럼이다.

사용자가 회고 내용을 작성한 후 수정했다면, `updatedAt`을 통해 마지막 변경 시점을 알 수 있다.

이를 활용하면:

* 최근 수정된 회고 목록 표시
* 오래 업데이트되지 않은 데이터 확인
* 사용자가 마지막으로 수정한 시간 표시

같은 기능을 구현할 수 있다.

또한 서비스 운영 과정에서 데이터 변경 이력을 확인하는 데 도움이 된다.

예를 들어 특정 데이터가 예상과 다르게 변경되었다면:

* 언제 수정되었는지
* 최근 변경된 데이터인지

를 확인하여 문제 원인을 찾을 수 있다.

---

### 데이터 정렬 및 최신 데이터 판단

`updatedAt`은 여러 데이터 중 **어떤 데이터가 최신 상태인지 판단하는 기준**으로도 사용된다.

예를 들어 여러 사용자가 같은 데이터를 수정할 수 있는 서비스에서는:

* 가장 최근에 수정된 데이터를 우선 사용
* 오래된 데이터를 최신 데이터로 덮어쓰지 않도록 방지

하는 기준이 될 수 있다.

---

### 자동 관리 방식

일반적으로 개발자가 직접 시간을 입력하지 않는다.

데이터베이스나 백엔드 프레임워크에서 자동으로 관리하도록 설정한다.

예를 들어 Spring JPA에서는:

* 데이터 생성 시 → `createdAt` 자동 저장
* 데이터 수정 시 → `updatedAt` 자동 변경

되도록 설정할 수 있다.

따라서 개발자가 매번 시간을 넣어주는 작업 없이 데이터의 생성·수정 시간을 일관되게 관리할 수 있다.
