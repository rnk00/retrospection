# 고민 주제

> **서비스를 배포한 후 사용자가 입력한 데이터는 어디에 저장될까? 개발자의 컴퓨터에 저장되는 걸까, 아니면 다른 곳에 저장되는 걸까? 또한 사용자가 많아지면 데이터는 어떻게 관리할까?**

---

# 결론

배포 후 사용자가 입력한 데이터는 **개발자의 컴퓨터가 아닌 클라우드 환경의 데이터베이스**에 저장된다.

실무에서는 대부분 **AWS RDS와 같은 데이터베이스 서비스**를 이용하며, 이미지와 같은 대용량 파일은 **AWS S3**에 별도로 저장한다.

사용자가 많아져도 데이터는 사용자 식별값(`user_id`)을 기준으로 관리하며, 데이터베이스와 파일 저장소를 분리하여 안정적으로 서비스를 운영한다.

---

# 배경

서비스를 개발하는 동안에는 MySQL을 자신의 컴퓨터에서 실행하며 데이터를 저장한다.

하지만 서비스를 배포하면 개발자의 컴퓨터는 계속 켜져 있지 않고, 사용자도 개발자의 컴퓨터에 접속하는 것이 아니다.

그렇다면 실제 서비스에서는 사용자가 작성한 데이터가 어디에 저장되는지, 또 많은 사용자가 동시에 서비스를 이용하면 데이터는 어떻게 관리되는지 궁금했다.

---

# 배포 후 데이터는 어디에 저장될까?

## 1. 클라우드 데이터베이스를 사용하는 방식

가장 일반적인 방법은 **클라우드에서 제공하는 데이터베이스 서비스**를 사용하는 것이다.

백엔드 서버(Spring Boot)는 이 데이터베이스와 연결되어 있으며, 사용자의 요청이 들어오면 데이터를 저장하거나 조회한다.

동작 과정은 다음과 같다.

1. 사용자가 웹 서비스에서 데이터를 입력한다.
2. Spring Boot 서버가 요청을 처리한다.
3. 데이터를 클라우드 데이터베이스(MySQL)에 저장한다.
4. 사용자가 다시 조회하면 데이터베이스에서 데이터를 읽어 화면에 보여준다.

대표적인 서비스로는 **AWS RDS**가 있다.

### 언제 사용할까?

* 실제 서비스 운영
* 협업 프로젝트
* 안정적인 데이터 관리가 필요할 때

### 장점

* 백업과 장애 복구를 지원한다.
* 데이터베이스 관리 부담이 적다.
* 성능과 확장이 용이하다.

### 단점

* 별도의 비용이 발생한다.

---

## 2. 가상 서버에 직접 데이터베이스를 설치하는 방식

클라우드 컴퓨터(예: AWS EC2)를 하나 빌린 뒤 그 안에 직접 MySQL을 설치하는 방법이다.

하나의 서버 안에서

* Spring Boot
* MySQL

을 함께 실행한다.

### 언제 사용할까?

* 개인 프로젝트
* MVP 개발
* 비용을 최소화하고 싶을 때

### 장점

* 비용이 저렴하다.
* 서버 구성이 단순하다.

### 단점

* 서버 자원을 함께 사용하기 때문에 성능 관리가 어렵다.
* 사용자가 증가하면 확장이 쉽지 않다.

---

# 사용자가 많아지면 데이터는 어떻게 관리할까?

## 1. 텍스트 데이터는 생각보다 용량이 크지 않다.

KPT 서비스에서 저장하는 대부분의 데이터는

* 회고 내용
* 제목
* 댓글

과 같은 텍스트이다.

텍스트 데이터는 이미지나 영상보다 용량이 매우 작기 때문에 초기 서비스에서는 수많은 데이터가 저장되더라도 큰 문제가 발생하지 않는다.

---

## 2. 이미지는 데이터베이스가 아니라 파일 저장소에 저장한다.

이미지를 데이터베이스에 직접 저장하면 데이터베이스 크기가 빠르게 증가한다.

그래서 실제 서비스에서는 이미지를 **AWS S3**와 같은 파일 저장 서비스에 저장한다.

동작 과정은 다음과 같다.

1. 사용자가 이미지를 업로드한다.
2. 이미지 파일은 S3에 저장된다.
3. S3가 생성한 이미지 주소(URL)를 데이터베이스에 저장한다.

즉, 데이터베이스에는 이미지 자체가 아니라 **이미지의 위치 정보(URL)** 만 저장한다.

### 장점

* 데이터베이스 크기가 불필요하게 커지지 않는다.
* 이미지 관리가 쉽다.
* 파일 다운로드 속도가 빠르다.

---

## 3. 여러 사용자의 데이터는 어떻게 구분할까?

실제 데이터베이스에는 모든 사용자의 데이터가 함께 저장된다.

하지만 각 데이터에는 **`user_id`** 와 같은 사용자 식별값이 함께 저장된다.

예를 들어 회고를 조회할 때는

```sql
SELECT *
FROM retrospect
WHERE user_id = ?;
```

처럼 로그인한 사용자의 `user_id`를 조건으로 조회한다.

따라서 하나의 데이터베이스를 여러 사용자가 함께 사용하더라도 자신의 데이터만 조회할 수 있다.

---

# 함께 알게 된 용어

### AWS RDS

AWS에서 제공하는 **관리형 데이터베이스 서비스**이다.

MySQL, PostgreSQL 등의 데이터베이스를 직접 설치하지 않고 사용할 수 있으며, 백업과 장애 복구 등을 AWS가 관리해 준다.

---

### AWS EC2

AWS에서 제공하는 **가상 서버**이다.

필요한 프로그램(Spring Boot, MySQL 등)을 직접 설치하여 사용할 수 있다.

---

### AWS S3

이미지, 영상, 파일과 같은 대용량 데이터를 저장하는 **객체 스토리지 서비스**이다.

실무에서는 이미지 파일 자체를 저장하기보다 데이터베이스에는 S3 URL만 저장하는 경우가 많다.
