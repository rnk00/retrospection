# 고민 주제

> **데이터베이스는 어떤 방식으로 생성할까? Entity를 만들면 자동으로 생성된다고 하는데, DDL을 직접 작성하는 방식과는 무엇이 다를까?**

---

# 결론

데이터베이스를 생성하는 방법은 크게 두 가지이다.

* **DDL(SQL)을 직접 작성하여 생성하는 방법**
* **JPA(Hibernate)의 `ddl-auto` 기능으로 자동 생성하는 방법**

개인 프로젝트에서는 `ddl-auto`를 사용하면 빠르게 개발할 수 있지만, 협업이나 운영 환경에서는 **DDL과 Flyway를 이용해 데이터베이스를 직접 관리하는 방식**이 일반적이다.

---

# 배경

Spring Boot 프로젝트를 시작하면 가장 먼저 데이터베이스를 준비해야 한다.

자료를 찾아보면

* Entity만 만들면 테이블이 생성된다고 하고,
* DDL을 직접 작성해야 한다고도 하며,
* Flyway를 사용해야 한다는 글도 볼 수 있다.

처음에는 이 방법들이 서로 다른 기술인 줄 알았지만, 실제로는 **데이터베이스를 생성하는 방식의 차이**였다.

---

# 데이터베이스를 생성하는 방법

## 1. DDL(SQL)을 직접 작성하는 방식

가장 전통적인 방법이다.

먼저 **ERD를 설계**한 뒤, 그 구조를 SQL(DDL)로 작성하여 데이터베이스를 생성한다. 이후 생성된 테이블 구조에 맞춰 Entity를 작성한다.

즉, **ERD → DDL → 데이터베이스 생성 → Entity 작성** 순서로 진행된다.

예를 들어 User 테이블을 만든다면 다음과 같이 SQL을 직접 작성한다.

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY,
    email VARCHAR(255)
);
```

### 언제 사용할까?

* 협업 프로젝트
* 운영 환경
* 데이터베이스 구조를 직접 관리하고 싶을 때

### 장점

* 데이터베이스 구조를 정확하게 관리할 수 있다.
* 변경 이력을 관리하기 쉽다.
* 협업과 유지보수에 유리하다.

### 단점

* SQL을 직접 작성해야 한다.
* 테이블 변경 시 `ALTER TABLE` 등을 직접 작성해야 한다.

---

## 2. Hibernate의 `ddl-auto`를 이용하는 방식

이번에는 Entity를 먼저 작성한다.

Hibernate는 Entity를 읽어 자동으로 `CREATE TABLE` SQL을 생성하고 실행한다.

즉, **Entity → Hibernate → SQL 생성 → 데이터베이스 생성** 순서로 진행된다.

설정은 다음과 같이 한다.

```properties
spring.jpa.hibernate.ddl-auto=create
```

### 언제 사용할까?

* 개인 프로젝트
* 프로토타입(MVP)
* 빠른 기능 개발

### 장점

* 개발 속도가 빠르다.
* SQL을 거의 작성하지 않아도 된다.

### 단점

* 생성되는 스키마를 세밀하게 제어하기 어렵다.
* 운영 환경에서 `create`나 `update`는 위험하다.
* ERD와 실제 데이터베이스 구조가 달라질 수 있다.

---

# 실무에서는 어떻게 할까?

실무에서는 대부분 **ERD를 먼저 설계**하고, 이를 바탕으로 **DDL을 작성**하여 데이터베이스를 생성한다.

이후 Flyway를 이용해 데이터베이스 변경 이력을 관리하고, 마지막으로 Entity를 작성하여 JPA와 연결한다.

즉, **DDL과 Flyway가 데이터베이스를 관리**하고, **JPA(Hibernate)는 이미 만들어진 데이터베이스를 자바 객체와 연결하는 역할**을 담당한다.

---

# 함께 알게 된 용어

### DDL(Data Definition Language)

데이터베이스 구조를 정의하는 SQL이다.

대표적으로 다음 명령어가 있다.

* `CREATE TABLE`
* `ALTER TABLE`
* `DROP TABLE`

---

### JPA

자바 객체와 데이터베이스를 연결하기 위한 **표준(API)** 이다.

개발자가 SQL 대신 객체 중심으로 개발할 수 있도록 도와준다.

---

### Hibernate

JPA를 실제로 구현한 라이브러리이다.

SQL 생성, CRUD 처리, 영속성 관리 등을 담당하며, `ddl-auto` 기능도 Hibernate가 제공한다.

---

### Flyway

데이터베이스 스키마 변경 이력을 관리하는 **마이그레이션 도구**이다.

예를 들어 컬럼을 추가하거나 테이블을 생성할 때 SQL 파일을 버전별로 관리하여 어떤 변경이 언제 적용되었는지 추적할 수 있다.

# 한 줄 정리

> **데이터베이스는 DDL을 직접 작성하거나 Hibernate의 `ddl-auto`를 이용해 생성할 수 있으며, 실무에서는 DDL과 Flyway로 스키마를 관리하고 JPA(Hibernate)는 데이터를 다루는 용도로 사용하는 것이 일반적이다.**
