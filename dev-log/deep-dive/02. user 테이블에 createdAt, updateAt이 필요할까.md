# 고민 주제

> **데이터를 저장할 때 `createdAt`과 `updatedAt` 컬럼은 왜 추가할까? 없어도 서비스는 동작하는데 꼭 필요한 걸까?**

---

# 결론

`createdAt`과 `updatedAt`은 서비스 동작에 반드시 필요한 컬럼은 아니지만, **데이터 관리와 서비스 운영을 위해 대부분의 프로젝트에서 함께 저장한다.**

* `createdAt`은 **데이터가 언제 생성되었는지** 기록한다.
* `updatedAt`은 **데이터가 마지막으로 언제 수정되었는지** 기록한다.

이 정보는 단순히 시간을 저장하는 것이 아니라, **기능 구현, 통계, 데이터 관리, 장애 분석** 등에 활용된다.

---

# 배경

회원가입, 게시글 작성, 회고 작성처럼 대부분의 서비스는 데이터를 계속 생성하고 수정한다.

처음에는 내용만 저장하면 된다고 생각했지만, 실제 서비스를 운영하다 보면 다음과 같은 기능이 필요해진다.

* 최신순으로 게시글을 보여주고 싶다.
* 이번 달 가입한 사용자 수를 알고 싶다.
* 사용자가 마지막으로 언제 수정했는지 표시하고 싶다.
* 문제가 발생한 데이터를 추적하고 싶다.

이러한 기능을 구현하기 위해 생성 시간과 수정 시간을 함께 저장한다.

---

# `createdAt`은 왜 필요할까?

`createdAt`은 **데이터가 생성된 시점**을 저장하는 컬럼이다.

### 활용 사례

* 회원가입 날짜 확인
* 게시글 작성 시간 표시
* 최신순 정렬
* 최근 7일 동안 작성된 회고 조회
* 이번 달 가입한 사용자 수 집계

또한 운영 중 문제가 발생했을 때도 유용하다.

예를 들어 특정 시점 이후 생성된 데이터에서만 오류가 발생했다면, 생성 시간을 기준으로 원인을 추적할 수 있다.

---

# `updatedAt`은 왜 필요할까?

`updatedAt`은 **데이터가 마지막으로 수정된 시점**을 저장하는 컬럼이다.

사용자가 회고 내용을 수정하면 수정될 때마다 시간이 갱신된다.

### 활용 사례

* 마지막 수정 시간 표시
* 최근 수정된 게시글 조회
* 오래 수정되지 않은 데이터 확인
* 최근 변경된 데이터 관리

또한 데이터가 예상과 다르게 변경되었을 때 언제 수정되었는지 확인하여 원인을 분석하는 데 도움이 된다.

---

# 최신 데이터를 판단하는 기준

`updatedAt`은 여러 데이터 중 **가장 최신 데이터를 판단하는 기준**으로도 사용된다.

예를 들어 여러 사용자가 하나의 데이터를 함께 수정하는 서비스에서는

* 가장 최근에 수정된 데이터를 우선 사용하거나,
* 오래된 데이터가 최신 데이터를 덮어쓰지 않도록

수정 시간을 기준으로 처리할 수 있다.

---

# 자동으로 저장되는 이유

일반적으로 개발자가 시간을 직접 입력하지는 않는다.

Spring Boot에서 JPA를 사용한다면 **JPA Auditing** 기능을 통해 자동으로 관리할 수 있다.

* 데이터를 처음 저장하면 `createdAt`이 자동으로 저장된다.
* 데이터를 수정하면 `updatedAt`이 자동으로 갱신된다.

덕분에 개발자가 직접 시간을 관리하지 않아도 항상 정확한 생성·수정 시간이 기록된다.

---

# 함께 알게 된 용어

### JPA Auditing

Entity의 생성 시간과 수정 시간을 자동으로 기록해 주는 JPA 기능이다.

`createdAt`과 `updatedAt`을 직접 설정하지 않아도 자동으로 관리할 수 있다.

---

### Timestamp

날짜와 시간을 함께 저장하는 데이터 형식이다.

`createdAt`과 `updatedAt`은 대부분 `TIMESTAMP` 또는 `DATETIME` 타입으로 저장된다.

---

### Entity

데이터베이스의 테이블과 대응되는 자바 클래스이다.

`createdAt`과 `updatedAt`은 Entity의 필드로 추가하여 관리한다.

---

# 추가로 공부할 내용

* JPA Auditing 설정 방법
* `@CreatedDate`, `@LastModifiedDate`
* Soft Delete와 `deletedAt`
* 시간대(Time Zone) 관리
* UTC와 Local Time

---

# 한 줄 정리

> **`createdAt`과 `updatedAt`은 단순히 시간을 기록하는 컬럼이 아니라, 서비스 기능 구현과 데이터 관리, 장애 분석을 위한 기본적인 메타데이터이다.**
